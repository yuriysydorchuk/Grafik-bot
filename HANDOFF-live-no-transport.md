# HANDOFF: «Лайв зміни» для фабрик без довозу (23.07.2026)

Разова записка сесії. Фікс хибного алярму на дашборді.

## Проблема

Віджет «Лайв зміни» (дашборд) показував для фабрик без довозу (AGRAM LUBLIN,
`factories.uses_transport = false`) водійські статуси: «очікує посадки», червоне
«посадку не підтверджено», «водій не призначений» — хоча посадки там не існує,
явку графікова ставить вручну. Лічильник `/attention` («зміни без водія») такі
фабрики вже пропускав — проблема була лише у `/live` + `LiveShifts.tsx`.

## Що зроблено

- **`routes/admin-api.ts`** — `GET /live` віддає `usesTransport` у кожній картці зміни.
- **`web/src/components/LiveShifts.tsx`** — для `usesTransport=false`:
  до початку — нейтральне «до початку X» (сіре); після початку без явок — жовте
  «явку не відмічено · до кінця X» (нагадування графіковій) замість червоного;
  з явками — зелене «на зміні», як було; у футері картки замість
  «водій не призначений» — «без довозу».
- **`web/src/lib/i18n.tsx`** — EN для нових ключів: «явку не відмічено» → "attendance
  not marked", «без довозу» → "no transport".

Фабрики з довозом не зачеплені. Кронові пресмінні/pickup-сповіщення й так
обходили фабрики без транспорту — бекенд-логіка не мінялася.

## Перевірено / деплой

- `pnpm run typecheck` — чисто; юніт-тести api-server — ок.
- Закомічено **окремо** від великого незакоміченого пласта (svodni2/payroll/ksef,
  sticky-шапки — ті лишаються в робочій копії, комітити після перевірки власником):
  свої ханки з `admin-api.ts`/`i18n.tsx` стейджились відфільтрованим патчем.
- Задеплоєно на прод 23.07.2026 (за явним підтвердженням власника).
