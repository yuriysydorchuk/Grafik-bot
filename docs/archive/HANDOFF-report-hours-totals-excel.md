## Що зроблено

Власник просив: рахувати **тотал годин по рапортах** і **ЗП по рапортах**; Excel з годинами рапорту — **польською**, з колонками код · ім'я · фабрика · подані години, гарно відформатований, зі **скачуванням по окремих фабриках**.

### API (`routes/admin-api.ts`)
- **`GET /hours`**:
  - У відповідь додано `totalReportHours` (сума `reportHours` по всіх працівниках) — не фінансове, видно всім з доступом до сторінки.
  - **owner-only**: на кожного працівника `reportNet`/`reportGross` = `calcPayroll(reportHours × ефективна ставка, isStudent, under26, rates)`; якщо рапорту нема — `null`. Плюс тотал `totalReportNet`.
  - Графікова ЗП (`net`/`gross` від відпрацьованих змін) лишилась як була — це **окремі** цифри від ЗП по рапорту.
- **`GET /hours/report-excel`** приймає опційний `?factoryId=`; назва файлу польська `Godziny raport <Фабрика> <YYYY-MM>.xlsx`. Хелпер тепер повертає `{ buffer, facName }`.

### Excel (`services/drive.ts` → `buildReportHoursExcel(month, factoryId?)`)
- Повертає `{ buffer, facName }` (раніше `Buffer`).
- **Польська**: заголовок `Godziny z raportu — [Фабрика —] <month pl>`; шапка `Kod · Imię i nazwisko · Fabryka · Godziny · Status`; статус `wysłano` / `brak` (жовтий для «brak»).
- Ім'я **разом** (одна колонка `Imię i nazwisko`) — окремих полів first/last у БД нема (лише `full_name`), і власник просив разом.
- Фільтр `factoryId` → лише працівники цієї фабрики; сорт польською.
- Рядок **`Razem`** із сумою поданих годин.
- Включає ВСІХ активних працівників (щоб видно було, хто не здав).

### Веб (`pages/Hours.tsx`, `lib/i18n.tsx`)
- Бейджі в шапці: **«Годин з рапорту»** (усім) та **«ЗП по рапорту»** (owner, синій).
- Owner-колонка **«ЗП по рапорту»** у таблиці (синім, щоб не плутати з графіковою ЗП) + сума у футері фабрики.
- У шапці кожної фабрики — кнопка **«Excel рапорту»** з `?factoryId=` (окремий файл на фабрику); глобальна кнопка в тулбарі лишилась.
- `Group` тепер несе `factoryId` (для лінка на факт-Excel). Нові i18n-ключі з EN-перекладами.

## Що знати наступній сесії
- **Шлях репо на проді — `/root/grafik-bot`** (нижній регістр), користувач `root`. pm2-процес `grafik-bot`.
- ЗП по рапорту рахується з **тієї ж ставки**, що й графікова (`effRate` → position/factory rate → worker rate → default). Різниця лише в кількості годин: рапорт = те, що працівник сам вписав; графік = з підтверджених змін.
- Фінансові колонки/тотали (`reportNet`, `totalReportNet`, ставка) — **лише owner** (`canFinance`), як і решта фінансів.
- Excel рапорту досі віддається стрімом через `/api/hours/report-excel` (не Google Drive) — Drive-токен для цього не потрібен.
