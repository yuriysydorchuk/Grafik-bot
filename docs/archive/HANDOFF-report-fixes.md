## 1) Рапорти не зберігалися — протермінований Google Drive токен (операційний фікс)

**Симптом:** працівники не могли здати рапорт; після фото — «❌ Помилка збереження».
**Корінь:** у логах прода `GaxiosError: invalid_grant` на кожній Drive-операції. Drive вантажиться **як реальний користувач** через OAuth ([getDriveAuth](../artifacts/api-server/src/services/drive.ts) — `GOOGLE_OAUTH_CLIENT_ID/SECRET/REFRESH_TOKEN`), і прод-`REFRESH_TOKEN` став недійсним. Ламало **всі** Drive-операції: рапорти, Excel-графік, облік годин, звіт поїздок.

**Що зроблено:**
- Згенерував новий `GOOGLE_OAUTH_REFRESH_TOKEN` через [`artifacts/api-server/get-google-token.mjs`](../artifacts/api-server/get-google-token.mjs) (`node --env-file=.env artifacts/api-server/get-google-token.mjs` → URL → консент у браузері акаунтом **yuriisydorchuk96@gmail.com** → токен у callback на `localhost:53682`).
- Вписав у **прод `.env`** (і локальний), з бекапами `.env.bak.*`. Прод читає env через `node --env-file=./.env` (ecosystem.config.cjs `node_args`), тож `pm2 restart` перечитує файл.
- **Durable fix:** власник опублікував OAuth-застосунок (Google Cloud Console → OAuth consent screen → **Publishing status = In production**). У статусі «Testing» Google протерміновує refresh-токени **кожні 7 днів** — це й була причина.
- **Перевидав токен ще раз** уже під опублікованим застосунком, щоб він не протермінувався (токени, видані в «Testing», вмирають за 7 днів навіть після публікації).

**Перевірка:** прямий тест токена — `OK Drive as: yuriisydorchuk96@gmail.com`; після рестарту `0 invalid_grant`; `healthz` 200.

**Якщо знову зламається:** повторити генерацію токена тим самим хелпером і вписати в прод `.env`. Перевірити, що OAuth-застосунок лишається **Published** (не Testing). `getDriveFolderLink` повертає `null` сам по собі (читає БД-налаштування `drive_root_folder_id`) — це НЕ індикатор авторизації; для перевірки авторизації роби прямий `drive.about.get`.

## 2) Працівники без підтверджених змін не могли здати рапорт (коміт `5b7d652`)

**Симптом:** «За місяць … у вас немає підтверджених змін. Зверніться до адміністратора.»
**Причина:** хендлер `bot.hears(menu.report)` ([bot/index.ts](../artifacts/api-server/src/bot/index.ts)) будував список фабрик **лише із затверджених `schedule_entries`** → у кого їх нема, впирався в глухий кут.
**Фікс:** кандидати фабрик = (затверджені зміни) **∪ власна фабрика працівника** (`workers.factoryId`). Одна → одразу фото; кілька → вибір. Якщо **ні затверджених змін, ні прив'язаної фабрики** — коректна помилка «Не вдалося визначити вашу фабрику для рапорту…» (за рішенням власника — **без** списку всіх фабрик).
**Не чіпали:** часове вікно подачі (останні/перші 7 днів місяця) лишилось як було.

> Наслідок для адмінів: якщо у працівника в картці не вказана фабрика — він побачить помилку. Сигнал проставити `factoryId` у профілі.
