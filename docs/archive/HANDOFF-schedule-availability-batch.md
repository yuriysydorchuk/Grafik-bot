## Що зроблено (по задачах)
- **T1 — графік видно лише після розсилки.** Нова колонка `schedule_entries.sent_at`. «Мій графік»
  (бот, `views.ts showWorkerSchedule`) показує лише записи з `sent_at`. Розсилка проставляє `sent_at`:
  веб `/schedule/notify` (тиждень) + новий параметр `day` (поденно); бот «Розіслати затверджений графік»
  (тиждень). У вебі (`Schedule.tsx`) — кнопка тижня (як була) + **під кожним днем** «Розіслати».
  `notifyFactorySchedule(weekId, weekStart, factoryId, day?)`.
- **T2 — аванс ≤ 500 zł** (`advance:enter_amount`, i18n `adv.maxAmount`).
- **T3 — «Зголосити відсутність» → «Взяти вихідний»** (5 мов: `menu.absence`, `abs.title`).
- **T4 — доступність лише адитивна.** Уже підтверджені зміни блокуються (`locked` у стані); confirm
  лише ДОДАЄ нові (не видаляє). Скасування — через «Взяти вихідний». i18n `av.locked`, оновлено `av.already`.
- **T5 — ручне додавання людини в будь-яку (навіть минулу/закінчену) зміну.** Пікер усіх активних
  працівників фабрики (`Schedule.tsx`), бо «Вільні» — лише з availability. Бекенд `/schedule/entry`
  вже дозволяв минуле.
- **T6 — Excel:** смуга-назва тепер на всю ширину (`bandCols = cols`) — суцільна кольорова лінія.
- **T7 — прибрано «📊 Читати таблицю»** (кнопка в `menus.ts` + обробник + мертвий `sheets:select_week`
  + невикористані імпорти).
- **T8 — заміна при відсутності лише з фабрики зміни** (бот, `absence:enter_reason`: фільтр
  `allWorkers` по `data.entryId`→factoryId). Веб уже фільтрував.
- **T9 — «не той тиждень»:** `getNextMonday/getCurrentMonday` рахували в UTC сервера → тепер
  `nowWarsaw()`. Нагадування й так повторювані (дедупу не було).
