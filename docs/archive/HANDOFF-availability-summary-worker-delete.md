## Що зроблено

### 1. Підсумок доступності біля назви фабрики
На сторінці **Доступність** під кожною фабрикою додано панель **«Розподіл по змінах»**:
- бейдж із загальною кількістю людей, що заповнили (вже був, біля назви);
- таблиця **День × Зміна** з лічильниками + колонка «Усього» по днях + рядок-підсумок по змінах.

**Балансування:** кожна людина рахується **рівно раз на день**. Хто вказав одну зміну — йде туди; хто вказав кілька — потрапляє жадібно в найменш завантажену зі своїх змін (спочатку найбільш «обмежені», тобто з найменшою кількістю варіантів), щоб колонки були плюс-мінус рівні. Уся логіка — **на фронтенді** (`buildSummary()`), API не змінювався.

### 2. Звільнені лишаються в списку + повне видалення (owner)
Звільнення/відновлення і таб «Звільнені» вже існували раніше. Додано **hard-delete**:
- **Бекенд:** `DELETE /workers/:id` під `requireRole("owner")`. У транзакції каскадно видаляє власну історію працівника (schedule_entries, availability, absence_requests, hours_disputes, worker_documents + фізичні файли через `deleteStoredFile`) і **занулює** посилання на чужих сутностях (absence_requests.substituteWorkerId, unplanned_workers.workerId, candidates.workerId/referrerWorkerId).
- **Фронтенд:** кнопка-кошик «Видалити назавжди» в рядку звільненого — видно **лише owner-у**, з підтвердженням `danger`. Звільнення лишає людину в списку; видаляє тільки кошик.
