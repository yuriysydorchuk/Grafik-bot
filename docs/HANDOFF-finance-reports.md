## Контекст

1. **Баг:** вкладка Фінанси показувала «пусто» за липень. `computeFinanceRange` фільтрував зміни за **понеділком тижня** (`weekStart >= '2026-07-01'`), тож тиждень Пн 29.06–Нд 05.07 цілком зараховувався червню (липень порожній, червень завищений). Модуль «Години» цю пастку вже обходив (`weekFromForMonth` + `entryDateStr`) — фінанси фікс оминув.
2. **Фіча:** тотали зарплат/доходів мають рахуватися в першу чергу по **рапортах** працівників (`monthly_reports`), і лише без рапорту — по затверджених явках.
3. **UI:** у картках зверху — окремі рядки ZUS до зарплати і ВАТ.

## Що зроблено (`routes/admin-api.ts`, `computeFinanceRange`)

- **Фактична дата:** тижні беруться з запасом −6 днів (`weekFromForMonth(start)`), кожна зміна фільтрується за `entryDateStr` у межах `[start, end)`. Виправляє і `GET /finance`, і `GET /finance/compare` (обидва рахують через цю функцію).
- **`hoursOverride`** тепер враховується (раніше фінанси його ігнорували — розсинхрон із вкладкою «Години»).
- **Агрегація** тепер per `(factory, worker, position, month)` — місяць у ключі потрібен для рапортних оверрайдів; payroll лінійний, тож розбивка точна.
- **Пріоритет рапортів:** для місяців, що **повністю** входять у діапазон (`fullMonthsIn`), рапорт worker+month+factory заміняє графікові години:
  - є графікові години → пропорційне масштабування порцій (зберігає розбивку по посадах/ставках);
  - графіка нема → години рапорту за поточною посадою/ставкою працівника (людина потрапляє і в лічильник «люди»);
  - легасі-рапорт без фабрики → фабрика, де людина має години того місяця (перевага поточній) — та сама логіка, що в `GET /hours`.
- **Нові поля API:** `salaryGross` (брутто ЗП) і `zusEmployer` (ZUS роботодавця) по фабриках і в `totals`; `laborCost` = їх сума, лишився для сумісності.

## Що знати наступній сесії

- **Правило «за фактичною датою зміни» тепер покриває і фінанси** — будь-який новий місячний зріз по `schedule_entries` мусить іти через `weekFromForMonth` + `entryDateStr`, а не по `weekStart`.
- Рапорти застосовуються **лише до повних календарних місяців** діапазону: режим порівняння «день-у-день» (mtd) — частковий місяць, лишається на явках (місячний рапорт не ділиться по днях). mom/yoy_month/yoy рапорти враховують.
- Рапорт за **поточний** (незавершений) місяць теж має пріоритет, якщо вже поданий — за вимогою власника.
- Фронт тепер жорстко читає `totals.salaryGross`/`zusEmployer` — старий бекенд + новий фронт = краш сторінки на `undefined.toLocaleString()`. Локально це і сталося: pm2 тримав старий dist. Після правок бекенду — `build` + `pm2 restart grafik-bot` (на проді не проблема: один процес віддає і API, і статику).
